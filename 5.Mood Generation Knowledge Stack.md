```markdown
# Design Vocabulary Reference

## Visual Aesthetics (11 core types)

### 1. Minimalist
- Characteristics: Lots of whitespace, 1–2 colors, sans-serif typography, clean lines

- Characteristics: Raw, unpolished, sharp shadows, geometric, no curves

- Typography: Monospace or blocky sans-serif (Space Mono)
- Imagery: Stark, high-contrast, documentary-style

- Typography: Monospace, stencil, or technical fonts
- Imagery: Mechanical, industrial, blueprints
- Spacing: Strict grid, mechanical alignment
- Use case: Engineering, tools, hardware

## Color Harmony Principles

### Monochromatic (One color family)
- Primary + 2–3 shades of that color
- Example: Navy (#001F3F) + Light blue (#0074D9) + Accent (#FF4136)

### Analogous (Adjacent on color wheel)
- 3 colors next to each other (e.g., orange, yellow-orange, yellow)
- Harmonious, warm or cool feeling

### Complementary (Opposite on color wheel)
- Two colors opposite each other (e.g., blue + orange)
- High contrast, energetic

### Triadic (Equidistant on color wheel)
- 3 colors equally spaced (e.g., red, yellow, blue)
- Vibrant, balanced, not used often in professional design

### Neutral-Forward
- 1–2 neutrals (black, white, gray) + 1–2 accent colors
- Professional, clean, safe
```

### Document B: Your Design System (Brutalist Aesthetic)

```markdown
# Ash's Design System — Brutalist Aesthetic

## Core Characteristics

You are NOT designing luxury (soft, refined). You are designing **brutalist** (raw, geometric, unapologetic).

### Typography
- **Display:** Space Mono (monospace, geometric, no-nonsense)
- **Body:** Inter (clean, modern, readable)
- **Color:** Black (#0A0A0A) on light backgrounds, white on dark
- **Spacing:** Tight, precise, grid-based
- **Line height:** Minimal for headings (1.1), normal for body (1.5)

### Visual Treatment
- **Border radius:** 0px (no curves, sharp edges only)
- **Shadows:** Harsh offset (4px 4px 0px black) — no soft gradients
- **Colors:** High contrast, limited palette (2–3 colors max)
- **Borders:** Sharp, 1–2px, dark color
- **Animations:** Fast, snappy, snap-to behavior (no floating)

### Spacing
- **Grid:** 8px base
- **Methodology:** Generous whitespace + abrupt transitions
- **No:** Rounded corners, soft shadows, gentle transitions

### Mood
- Raw, unpolished, geometric, honest
- Like a design manifesto carved in concrete
- Not cute, not luxury, not playful
- Professional rebellion: serious but unconventional

## When Generating Moods:

**DO NOT generate moods that contradict your brutalist core:**
- ❌ Soft pastels + rounded corners (that's Playful/Toy-like, not you)
- ❌ Luxury elegance with lots of spacing (too refined for brutalism)
- ❌ Illustrated cuteness (not your vibe)

**DO generate moods that respect brutalism:**
- ✅ Bold & Data-Driven (brutalist fits this perfectly)
- ✅ Minimalist with sharp edges (your core)
- ✅ Typographic-heavy with Space Mono (your DNA)
- ✅ Industrial/utilitarian (fundamentally brutalist)
- ✅ Dark mode with harsh shadows (your style)

**Your 3 moods should be distinct BUT all respect brutalism.**
Example: Don't generate "Luxury Elegant" as one of the 3 options. Generate "Bold & Data-Driven", "Typographic Minimalist", "Industrial Dark"—all brutalist variants with different emphasis.
```

## Layer 4: Mood Generation Logic (Implementation)

### How the API Endpoint Works

```typescript
// functions/index.js (Cloudflare Worker)

async function generateMoods(userInput: string) {
  // 1. Load research documents (these are strings in the worker)
  const designVocabulary = DESIGN_VOCABULARY_REFERENCE; // Doc A
  const designSystem = ASH_DESIGN_SYSTEM;               // Doc B
  const existingPresets = EXISTING_PRESETS_JSON;        // Doc C

  // 2. Construct the full prompt
  const prompt = `
    ${MOOD_GENERATION_SYSTEM_PROMPT}
    
    User Input: "${userInput}"
    
    [Full Document A: Design Vocabulary...]
    ${designVocabulary}
    
    [Full Document B: Your Design System...]
    ${designSystem}
    
    [Full Document C: Existing Presets to avoid...]
    ${existingPresets}
  `;

  // 3. Call Claude API with max_tokens high enough for JSON response
  const response = await fetch("https://api.anthropic.com/v1/messages", {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": CLAUDE_API_KEY
    },
    body: JSON.stringify({
      model: "claude-3-5-sonnet-20241022",
      max_tokens: 2000,  // Enough for 3 moods + metadata
      messages: [
        {
          role: "user",
          content: prompt
        }
      ]
    })
  });

  const data = await response.json();
  
  // 4. Extract JSON from response
  const content = data.content[0].text;
  const jsonMatch = content.match(/\{[\s\S]*\}/);
  const moods = JSON.parse(jsonMatch[0]);

  // 5. Validate and return
  return {
    moods: moods.moods,
    timestamp: new Date().toISOString(),
    promptTokens: data.usage.input_tokens,
    completionTokens: data.usage.output_tokens
  };
}
```

## Layer 5: Why Your Brutalist Aesthetic Matters

Here's the critical connection you were asking about:

```
Claude's general knowledge of design

Design Vocabulary Reference (11 aesthetic types)
       ↓
```

- ❌ "Luxury Elegant" (soft shadows, serif elegance) — contradicts rawness

### With Your Design System

Claude knows:
- ✅ Space Mono is the display font (no other option)
- ✅ Shadows are harsh offsets, not soft glows
- ✅ No border radius means all shapes are geometric
- ✅ High contrast is non-negotiable
- ✅ Brutalism = rebellion + precision

So it generates:
- ✅ "Bold Data-Driven" (navy + orange, Space Mono, sharp shadows)
- ✅ "Minimalist Fitness" (white + black + green, geometric, breathing space)
- ✅ "Dark Industrial" (dark bg + neon blue, grid-based, mechanical)

All 3 are **contextually relevant to fitness dashboards** AND **respect your aesthetic DNA**.

## Summary: How Moods Get Generated

| Layer | What | Example |
|-------|------|---------|
| **1. Knowledge** | Claude knows design theory, color harmony, aesthetics | "Complementary colors create energy" |
| **2. Research Docs** | We provide: Design vocabulary + your aesthetic + existing presets | "Brutalism means no curves, Space Mono always" |

| **4. Logic** | Claude outputs JSON with mood name, colors, fonts, keywords | "Bold Data-Driven: #001F3F + Space Mono" |
| **5. Your Aesthetic** | All output is filtered through your design system DNA | All 3 moods respect: no border-radius, harsh shadows, Space Mono |
