## PHASE 2B: AI MOCKUP GENERATION (DAYS 5-6, 12 HOURS)
### Goal: Generate unique AI mockups using Flux 2 Pro + validation loop

### What You're Building

**This is the magic that solves commoditization.**

Flow:
```
Unsplash images analyzed (Phase 2) + 5 Typography options selected
  ↓
Claude writes detailed visual prompt for each section
  ↓
Flux 2 Pro generates image:
  - Text prompt (visual description)
  - Unsplash reference images (up to 8)
  - Aspect ratio (16:9 for hero, etc.)
  ↓
Claude Vision validation (NEW):
  ├─ Does mockup match mood aesthetic?
  ├─ Is text readable?
  ├─ Is quality acceptable?
  ├─ Pass → Show to user
  └─ Fail → Regenerate (max 2 retries)
```

**Why Flux 2 Pro:**
- Handles complex text rendering + photorealistic details
- Maintains consistent characters/styles across multiple reference images (up to 8)
- Text + image input together = guided generation from references
- Unsplash images inform style/aesthetic, not just copy-pasted
- Result: Completely unique mockups that match mood + brand direction

**Validation Loop (NEW - Claude Vision Check):**
```typescript
async function validateAndReturnMockup(imageUrl, mood, section, retryCount = 0) {
  // Use Claude Vision to validate the generated mockup
  const validation = await claude.messages.create({
    model: "claude-opus-4-20250805",
    max_tokens: 300,
    messages: [
      {
        role: "user",
        content: [
          {
            type: "image",
            source: { type: "url", url: imageUrl },
          },
          {
            type: "text",
            text: `Validate this "${section}" mockup for brand "${mood.name}". Check:
1. Is text readable and legible?
2. Does the aesthetic match mood: ${mood.description}?
3. Is the layout coherent and professional?
4. Are colors visible and well-applied?

Return JSON: { isValid: boolean, issues: [...], suggestion: "..." }`
          }
        ],
      }
    ],
  });

  const result = JSON.parse(validation.content[0].text);
  
  if (result.isValid) {
    return imageUrl; // Pass - return to user
  }
  
  if (retryCount < 2) {
    // Regenerate with adjusted prompt
    const adjustedPrompt = `${visualPrompt}. ${result.suggestion}`;
    const retryResponse = await replicate.run("black-forest-labs/flux-2-pro", {
      input: {
        prompt: adjustedPrompt,
        image: unsplashImageUrls,
        aspect_ratio: "16:9",
      },
    });
    return validateAndReturnMockup(retryResponse[0], mood, section, retryCount + 1);
  }
  
  // Max retries exceeded - return image anyway with warning
  console.warn(`Mockup validation failed after 2 retries for ${section}`);
  return imageUrl;
}
```

## PHASE 2D: INTERACTIVE REFINEMENT LOOP - FLUX KONTEXT MAX (ONGOING)
### Goal: Let user iterate and refine mockups non-destructively

**Flux Kontext Max = Image editing model (NOT generation)**

Users can refine existing mockups:

1. **Edit text:**
   - "Change headline to 'Premium Fitness App'"
   - Flux Kontext Max understands text in images, edits it, preserves rest
   - Non-destructive: background, layout, design stay intact

2. **Change colors (with re-validation):**
   - "Change primary color from navy to teal across all mockups"
   - Flux Kontext Max applies color changes
   - Claude Vision re-validates each mockup after color change
   - All mockups propagate the new color palette

3. **Modify specific elements:**
   - "Make the background lighter"
   - "Remove this element"
   - "Swap the hero image"
   - Kontext Max preserves everything else

4. **Generate 3 variations:**
   - "Show me 3 versions of the hero with different layouts"
   - Use Flux 2 Pro (generation) not Kontext (editing)
   - User picks favorite

5. **Compare and merge:**
   - Keep previous versions
   - "I like logo from v2 and website from v3"
   - Build final design from best pieces

**Implementation:**

New components:
- `RefinePromptPanel.tsx` - Text input for refinements
- `VariationComparison.tsx` - Side-by-side comparison
- `ColorEditor.tsx` - Change colors + re-validate
- `TextEditMode.tsx` - Direct text editing on mockup

**API Endpoints:**
```
POST /api/refine-mockup
  Input: { mockupUrl, editInstruction, mockupId }
  → Flux Kontext Max edits image
  → Claude Vision validates output
  → Return edited image

POST /api/generate-variations
  Input: { visualPrompt, unsplashImages, count }
  → Flux 2 Pro generates (NOT Kontext)
  → Validate each
  → Return 3 images

POST /api/propagate-colors
  Input: { newPalette, allMockupIds }
  → For each mockup:
    - Flux Kontext Max: recolor with new palette
    - Claude Vision: validate
  → Return all updated mockups
```
