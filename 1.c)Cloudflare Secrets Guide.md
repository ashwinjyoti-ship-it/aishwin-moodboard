# Where to Store REPLICATE_API_TOKEN
## Workers vs Pages vs wrangler.toml

---

## Your Current Setup

Based on your repo structure:
- **Frontend:** React + Vite → deployed to Cloudflare **Pages**
- **Backend:** `functions/index.js` → deployed as Cloudflare **Workers**
- **Database:** D1 (SQLite)

---

## Where the Token Goes

### Answer: **Cloudflare Workers Secrets**

The token is used in your **backend** (`functions/index.js`), not your frontend.

Since `functions/index.js` is a Cloudflare Worker, you add the secret there.

---

## How to Add It

### Option 1: Via Cloudflare Dashboard (Easiest)

1. **Go to:** https://dash.cloudflare.com
2. **Navigate:** Workers & Pages → your project → Functions
3. **Settings → Environment Variables**
4. **Add variable:**
   - Name: `REPLICATE_API_TOKEN`
   - Value: `r8_xxxxxxxxxxxx` (your Replicate token)
5. **Save**

### Option 2: Via wrangler.toml (Recommended for Teams)

**File: `wrangler.toml`**

```toml
name = "mood-board"
type = "service-worker"
main = "functions/index.js"

# Environment variables
[env.production]
vars = { REPLICATE_API_TOKEN = "r8_xxxxxxxxxxxx" }

[env.development]
vars = { REPLICATE_API_TOKEN = "r8_test_xxxxxxxxxxxx" }
```

**Deploy with:**
```bash
wrangler deploy --env production
```

### Option 3: Via GitHub Secrets (For CI/CD)

**If using GitHub Actions to deploy:**

1. Go to: GitHub repo → Settings → Secrets and variables → Actions
2. Create new secret: `REPLICATE_API_TOKEN`
3. In `wrangler.toml` reference it:
```toml
[env.production]
vars = { REPLICATE_API_TOKEN = "${REPLICATE_API_TOKEN}" }
```

---

## Important: DO NOT Put in Pages Secrets

**Pages secrets are for your frontend.**

Your frontend (React) running in the browser can't access Pages secrets safely. 

**Your Flux code runs on the Worker (backend), so use Workers secrets.**

---

## Where Each Secret Goes

| Secret | Goes In | Why |
|--------|---------|-----|
| `REPLICATE_API_TOKEN` | **Workers** | Used in `functions/index.js` (backend) |
| `CLAUDE_API_KEY` | **Workers** | Used in Worker endpoints (backend) |
| `UNSPLASH_API_KEY` | **Workers** | Used in Worker endpoints (backend) |
| `D1_DATABASE_ID` | **Workers** | Used in Worker to access D1 |
| Frontend config (theme, etc.) | **Pages** | Used in React frontend |

  auth: replicateToken,
});

## Common Mistakes

❌ **Mistake 1: Adding to Pages Secrets**
- Pages secrets are for frontend
- Your code runs on Workers (backend)
- Add to Workers instead

❌ **Mistake 2: Hardcoding token in code**
- Never commit `REPLICATE_API_TOKEN` to GitHub
- Always use environment variables
- `.gitignore` your local `.env` files

❌ **Mistake 3: Using wrong environment**
- Production and development can have different tokens
- Use `wrangler.toml` with `[env.production]` and `[env.development]`
- Deploy to correct environment
